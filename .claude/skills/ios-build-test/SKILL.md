# iOS build — BrickScan

This project's `.xcodeproj` is generated by xcodegen from `project.yml`. Never edit
`BrickScan.xcodeproj` or `BrickScan/Resources/Info.plist` by hand — both are regenerated and
your edits will be silently lost. Edit `project.yml` instead.

There is **no test target** — removed deliberately (personal app, the suite covered almost
nothing and wasn't worth the upkeep). Don't add `Tests/`/`BrickScanTests` back, or a test step
to this workflow, without being asked. Verify behavior manually (skill `verify`/`run`) instead.

## Steps

1. If you added/removed/moved any source file, or edited `project.yml`:
   ```bash
   xcodegen generate
   ```

2. Build:
   ```bash
   xcodebuild -project BrickScan.xcodeproj -scheme BrickScan \
     -destination 'generic/platform=iOS Simulator' build 2>&1 | grep -E "error:|BUILD"
   ```
   Fix every `error:` before moving on. Warnings are not blocking.

3. Only report the change as done once build prints `** BUILD SUCCEEDED **` with no `error:`
   lines.

## Don't

- Don't pass `--no-verify`-equivalent shortcuts; there are none here, just don't skip steps.
- Don't run the full build log unfiltered into context — always pipe through grep as shown; the
  raw xcodebuild log is tens of thousands of lines.
