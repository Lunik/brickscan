# iOS build & test — BrickScan

This project's `.xcodeproj` is generated by xcodegen from `project.yml`. Never edit
`BrickScan.xcodeproj` or `BrickScan/Resources/Info.plist` by hand — both are regenerated and
your edits will be silently lost. Edit `project.yml` instead.

## Steps

1. If you added/removed/moved any source file, or edited `project.yml`:
   ```bash
   xcodegen generate
   ```

2. Build:
   ```bash
   xcodebuild -project BrickScan.xcodeproj -scheme BrickScan \
     -destination 'generic/platform=iOS Simulator' build 2>&1 | grep -E "error:|BUILD"
   ```
   Fix every `error:` before moving on. Warnings are not blocking.

3. Run tests:
   ```bash
   xcodebuild test -project BrickScan.xcodeproj -scheme BrickScan \
     -destination 'platform=iOS Simulator,name=iPhone 17' 2>&1 \
     | grep -E "error:|Test Suite 'All|\*\* TEST"
   ```

   If this fails with `Invalid device state`, `Mach error`, `Failed to create a bundle
   instance...`, or similar simulator-runtime errors (NOT a compile or assertion error), the
   simulator is in a bad state, not your code. Reset and retry once:
   ```bash
   xcrun simctl shutdown all
   xcodebuild test -project BrickScan.xcodeproj -scheme BrickScan \
     -destination 'platform=iOS Simulator,name=iPhone 17' 2>&1 \
     | grep -E "error:|Test Suite 'All|\*\* TEST"
   ```
   If a specific simulator UDID is consistently broken, `xcrun simctl erase <udid>` before
   retrying (last resort — wipes that simulator's state).

   Use `xcrun simctl list devices available` to find another booted/available iPhone
   destination by name if `iPhone 17` isn't present on this machine.

4. Only report the change as done once **both** build and test print success
   (`** BUILD SUCCEEDED **` / `** TEST SUCCEEDED **` with no `error:` lines). Don't rely on
   build success alone — this project has caught real regressions (decoding bugs, concurrency
   issues, UI-update bugs) only at test time.

## Don't

- Don't add `-skip-testing` or filter out failing tests to get to green — fix the failure.
- Don't pass `--no-verify`-equivalent shortcuts; there are none here, just don't skip steps.
- Don't run the full test suite output unfiltered into context — always pipe through grep as
  shown; the raw xcodebuild log is tens of thousands of lines.
